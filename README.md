# InsightForge

Sales forecasting and interactive dashboard for the Superstore dataset. InsightForge contains exploratory data analysis (EDA), time-series feature engineering, model training and evaluation (Linear Regression, Random Forest, Gradient Boosting), and a lightweight HTML dashboard that visualizes historical sales and a 6‑month forecast.

## What you'll find here

- sales_forecasting.ipynb / sales_forecasting_complete.ipynb — Jupyter notebooks with EDA, feature engineering, model training, evaluation, and visualizations.
- train_export_forecast.py — Script to train models (walk-forward validation), choose the best model by MAPE, fit a final model and export forecast.json and metrics.json. Also optionally serializes model.joblib when joblib is installed.
- index.html — Static dashboard that loads exported forecast/metrics and shows interactive charts and insights.
- Sample - Superstore.csv — Example Superstore dataset used by the notebooks and script.
- model.joblib — (Optional) serialized model produced by the training script.
- forecast.json, metrics.json — Outputs produced by running train_export_forecast.py.
- images (.png) — Visual assets generated by the notebooks (EDA, feature importance, forecast charts, seasonality heatmap).
- LICENSE — Repository license (MIT).

## Quickstart (run locally)

1. Create a virtual environment and install dependencies (minimum):

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install pandas numpy scikit-learn matplotlib seaborn joblib openpyxl
```

2. Place the Superstore CSV in the repository root (file already included as `Sample - Superstore.csv`) and run the training/export script:

```bash
python train_export_forecast.py
```

This will produce `forecast.json` and `metrics.json` and (if joblib is available) `model.joblib`.

3. View the dashboard locally by opening `index.html` in a browser, or serve the directory with a static server:

```bash
# from the repo root
python -m http.server 8000
# then open http://localhost:8000 in your browser
```

Notes:
- The notebooks include reproducible EDA and plotting steps and are a good starting point to explore intermediate outputs.
- The training script expects the CSV to contain `Order Date` and `Sales` columns; it aggregates data to monthly sales and creates lag/rolling seasonality features.

## Files of interest (short)

- `train_export_forecast.py` — full pipeline for walk-forward validation, model selection by MAPE, and iterative forecasting with optional bootstrap intervals.
- `sales_forecasting_complete.ipynb` — annotated notebook showcasing EDA, model comparison and visualizations (creates eda_overview.png, forecast_vs_actual.png, feature_importance.png, seasonality_heatmap.png, etc.).
- `index.html` — polished static dashboard UI that reads exported JSON and displays charts and KPI cards.
